## 1. 背景・位置づけ

- 90分ハンズオン（IT 20分／数値AI 35分／生成AI 35分）の **第3パート＝生成AI（35分）**。
- ねらい（スライド56・61）：**要件定義 → 仕様駆動コーディング → 調整** の開発プロセスに沿えば、**自分でも形にできる**を体感する。
- ITで作った「押す→処理が走る」を、ここでは **[生成ボタン]＝要件md → コード生成** として使い回す（一本の糸）。
- 体験のねらい（スライド61）：
  1. 開発プロセスに沿えば自分で作れる（要件md→コーディング→調整で形になる）
  2. 同じ流れは**自部署課題に応用できる**
  3. 作った後も好みに調整できる ＝「自分で作れる」を持ち帰る

### 1.1 他2パートとの「作り方」の違い（重要・FB反映）

| パート | 参加者の操作 |
|---|---|
| IT / 数値AI | **アプリ内編集セル**で最小の穴埋め（書く体験） |
| 生成AI（本書） | **事前作成のまーぴーを、Claude Code でバイブコーディング**してアレンジ（3Step） |

→ 生成AIは「一から作る」のではなく「**用意された動くまーぴーを自分流に変える**」。気軽に手を動かし、AIに任せて形にする体験（バイブコーディング）。

## 2. 目的・ゴール

- **基本路線＝事前作成の「まーぴー」を全員でアレンジする**（音声で話せるbot）。
- **連結＝まーぴーが数値AIの予測Ad数を「話す」**（一本の糸を体感。ベスト：推論エンドポイントを呼ぶ／ベター：出力CSVを読む）。
- **応用＝同じ気軽さ（バイブコーディングで少し変える）を自部署課題に当てはめられる**、という見せ方。
- ゴール：参加者が「動くまーぴーを **3Stepで自分流に変える**（方言・多言語・動き）」を体験し、「自分でも変えられる／作れる」を持ち帰る。
- ※まーぴーがうまく動かない場合は **ライオンちゃん**（別キャラ）で代替してよい。

## 4. 当日フロー（バイブコーディング・3Step／FB反映）

事前作成の動くまーぴーを起点に、**3Stepで自分流にアレンジ**する。一から作らない。

| Step | 内容 | やること |
|---|---|---|
| 事前 | （講師）最低限のまーぴーアプリを用意・デプロイ | 参加者は動く状態から始める（🔒） |
| Step1 | 動かしてみる | 配布URL/環境でまーぴーと話す → 何ができるか掴む（🔒 体験） |
| Step2 | 1つアレンジする | Claude Code に「方言にして」「英語も話せるように」等を**話し言葉で頼む**（🟢 バイブコーディング） |
| Step3 | もう一声・連結 | 動き（アニメ）を足す／**数値AIの予測Ad数を話させる**（🟢 連結体験） |

> 「仕様書を書く」のではなく、**やりたいことをClaude Codeに言って変えてもらう**のがバイブコーディング。
> 講師は裏で「まーぴー仕様（md）」を題材に、何が起きているかを解説する。

## 7. 配布物の要件（FB反映：バイブコーディング用ガイド）

### 7.1 アレンジ用カスタマイズガイド（配布md）

- 一から作る要件テンプレではなく、**動くまーぴーに「こう変えて」とClaude Codeへ頼む例集**。
- 3Stepで完結する**話し言葉プロンプト例**を用意（コピペで使える）。例：
  - 方言：「まーぴーの口調を関西弁にして」
  - 多言語：「英語でも話せるようにして」
  - 動き：「話すときにもっと大きく跳ねるようにして」
  - **連結**：「数値AIの予測Ad数を聞かれたら、それを読み上げて教えて」
- 🔒触らない骨格（音声接続・短命トークン・WebRTC）と、🟢変えてよい部分（口調・言語・見た目・連結）を明示。

### 7.2 自部署課題 応用テンプレ（md）

- まーぴーと同じ構成（**タイトル／目的／入力／処理／出力／制約**）を**埋めるだけ**のスケルトン。
- 「まーぴーで掴んだ気軽さ（言って変える）を、自分の業務課題に当てはめる」見せ方。

### 7.3 数値AI連結（FB反映・一本の糸）

- **当日方針（確定 2026-06-19）：まず「ベター（CSV読み）」を確実に。**
  まーぴーが**数値AIの出力CSV（予測Ad数）を読み込み**、聞かれたら音声で答える。
- 「ベスト（推論エンドポイント直呼び）」は余力があれば差し替え。
- 連結の実装は数値AI側の出力CSV（仕様設計書 §13）とセット。

## 9. 受入基準（HANDOFF §6.6 準拠）

- [ ] 参照まーぴーアプリが `streamlit run` で起動し、マイク許可後に**音声で対話**できる
- [ ] 接続後にまーぴー（青い小鳥）が表示され、**発話に合わせて動く**
- [ ] ユーザー/まーぴー双方の**文字起こし**が表示される
- [ ] **プロンプト1箇所**の編集で口調が変わる（FR-08）
- [ ] ブラウザ通信に**永続APIキーが含まれない**（短命トークンのみ：FR-02）
- [ ] **最低限のまーぴーが事前にデプロイ済**で、参加者は動く状態から始められる
- [ ] **3Stepのバイブコーディング**で、方言・多言語・動き等のアレンジが実際にできる
- [ ] まーぴーが**数値AIの予測Ad数を話せる**（ベスト：エンドポイント／ベター：CSV）
- [ ] カスタマイズガイド（頼み方の例集）と自部署課題テンプレが揃っている
- [ ] `main.py` のキャラが**まーぴー（青い小鳥）に統一**されている（わんく残存なし）。※ダメな場合はライオンちゃん差し替え可
